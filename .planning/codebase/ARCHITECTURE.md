# Architecture

## App Structure
Single-page application (SSR-enabled) with a single route at `/`.

```
src/
├── start.ts              # TanStack Start instance + SSR error middleware
├── server.ts             # Server entry — SSR error boundary wrapper around @tanstack/react-start/server-entry
├── router.tsx            # Router factory (QueryClient + createRouter)
├── routeTree.gen.ts      # Auto-generated route tree (TanStack Router file-based routing)
├── styles.css            # Tailwind v4 + design tokens (oklch colors, dark mode)
├── hooks/
│   └── use-mobile.tsx    # Mobile breakpoint hook (768px)
├── lib/
│   ├── conductor-data.ts # Models, role presets, mock response generator
│   ├── utils.ts          # cn() utility (clsx + tailwind-merge)
│   ├── error-capture.ts  # Global error/ rejection capture for SSR
│   ├── lovable-error-reporting.ts  # Lovable SDK error bridge
│   └── error-page.ts     # Static error page HTML renderer
├── components/
│   ├── conductor/
│   │   └── AddParticipantModal.tsx  # Add participant dialog
│   └── ui/               # 46 shadcn/ui components (Radix-based)
├── routes/
│   ├── __root.tsx        # Root layout (shell, QueryClientProvider, Toaster, 404/error boundaries)
│   └── index.tsx         # MAIN: Conductor page — ALL app logic inline
└── start.ts / server.ts  # SSR entry points
```

## Conductor Page (routes/index.tsx — 582 lines)

### Types (inline)
- `Participant` — id, model, roleName, systemPrompt, isEnabled
- `Message` — id, sender (user|agent), authorName, roleName, model, content, streaming, tokens
- `Session` — id, title, createdAt, participants[], messages[]

### State (all in one component)
- `sessions` — array of sessions (useState)
- `activeId` — currently active session ID (useState)
- `input` — text input value (useState)
- `autoRun` — auto-run toggle (useState)
- `addOpen` — add participant modal state (useState)
- `autoRunRef` + `streamingRef` — useRef flags for async safety
- `messagesEndRef` — scroll anchor ref

### Actions
- `newSession()` — creates a seed session with 2 default participants (Optimistic Entrepreneur + The Skeptic)
- `deleteSession(id)` — removes session, creates new if last
- `sendUserMessage()` — appends user message, sets session title from first message
- `streamAgentResponse(participantId)` — mock streaming via setInterval (180ms per chunk)
- `runAutoLoop()` — iterates enabled participants, calls streamAgentResponse sequentially
- `toggleAutoRun()` — start/stop auto-run loop
- `triggerParticipant(id)` — manually trigger single agent
- `toggleParticipant(id, v)` — enable/disable participant
- `addParticipant(data)` — adds new participant via modal callback
- `removeParticipant(id)` — removes participant

### UI Components (generated by shadcn/ui CLI)
46 components in `components/ui/` — standard Radix-based shadcn wrappers. All are present but many are unused in current code.

## Server/SSR Architecture
1. `start.ts` creates TanStack Start instance with error middleware
2. `server.ts` SSR entry with h3 error swallowing detection
3. `error-capture.ts` captures global errors/unhandled rejections with 5s TTL
4. `error-page.ts` renders static HTML error page
5. `lovable-error-reporting.ts` bridges errors to Lovable dev tools
6. `__root.tsx` includes `errorComponent` for client-side error boundaries

## Known Gaps
- No API connector — mock data only
- No orchestrator/state machine — sequential dispatch only
- No session persistence (in-memory state, lost on refresh)
- No real LLM streaming — setInterval-based mock
- react-hook-form, zod installed but unused (AddParticipantModal uses raw state)
